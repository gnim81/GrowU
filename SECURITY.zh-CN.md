# 安全策略

## 支持的版本

安全更新适用于最新的公开发布版本。如果您在正式发布前的默认分支上发现漏洞，请通过相同的非公开流程报告。

## 报告漏洞

对于疑似漏洞，请勿在 GitHub 上创建公开 Issue。

请使用 GitHub 的非公开漏洞报告功能，或为该仓库起草一份安全公告（若该功能可用）。如果仓库后续列出了专门的安全联系人，请改用该渠道。如果没有非公开渠道可用，请通过仓库页面上所列的最不公开的渠道联系维护者，并在确认非公开路径之前不要包含漏洞利用细节。

请在报告中包含以下信息：

- 受影响版本，如有可能请提供确切的提交号
- 部署模式，例如 Docker Compose 或手动部署
- 清晰的复现步骤、验证概念或触发输入
- 预期影响，包括数据泄露、权限提升或服务中断风险

请仅提供最少量的数据以安全地复现问题。
