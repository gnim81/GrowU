# GrowU V1 规划

## 产品目标

GrowU V1 是一个响应式 Web 应用，用于追踪孩子的正向强化和基于积分的进度。它面向希望用简单方式记录奖励事件、惩罚事件、兑换奖励、交易历史、编辑记录、统计数据以及 CSV 导出的家长或监护人，同时保留历史记录。

## 功能范围

- 基于数据库的用户账户
- 首次运行通过 `/setup` 设置管理员
- 在 `/login` 登录
- 在 `/settings/accounts` 管理管理员账户
- 孩子档案管理
- 奖励、惩罚和兑换项目管理
- 积分交易创建
- 含余额检查的奖励兑换
- 含修订审计历史的交易编辑
- 统计数据和 CSV 导出
- 移动端和桌面端兼容的 UI

## 不包含的功能

- 孩子自助账户
- 多家庭或多租户支持
- 离线模式
- 推送通知
- 附件或证据上传
- 自动定期重置
- 超出文档化自托管流程的部署自动化

## 技术基础

- Next.js App Router
- TypeScript
- Tailwind CSS
- Prisma
- PostgreSQL
- 标准 Node.js 部署或基于 Docker 的自托管

## 当前账户模型

GrowU 现在通过 `UserAccount` 将账户存储在数据库中。

规则：

- 全新安装通过 `/setup` 创建首个管理员
- 用户在 `/login` 登录
- 管理员在 `/settings/accounts` 管理账户
- 必须始终保持至少一个启用的管理员账户
- `GROWU_ACCOUNTS` 仅作为升级时的旧版导入源存在

## 数据规则

### `UserAccount`

- 存储登录身份、显示名称、密码哈希、角色和启用状态
- 支持 `ADMIN` 和 `PARENT` 角色
- 必须保留至少一个启用的管理员

### `Child`

- 孩子可以创建、编辑和禁用
- 正常的产品流程不会硬删除孩子
- 孩子被禁用后，历史交易必须保持可见

### `PointItem`

- 奖励、惩罚和兑换项目可以创建、编辑和禁用
- 正常的产品流程不会硬删除项目
- 已禁用的项目应从新条目选择器中消失，但在历史记录中仍有意义

### `PointTransaction`

- 存储所有积分变化
- 使用整数值表示积分
- 奖励交易增加余额
- 惩罚交易减少余额，可降至零以下
- 兑换交易减少余额，余额不足零时操作必须失败
- 历史项目显示依赖 `itemNameSnapshot`

### `TransactionRevision`

- 存储编辑前后的快照
- 记录原因、编辑者和时间戳
- 提供交易变更的审计追踪

## 功能验收标准

### 认证与设置

- 当没有账户存在时，`/setup` 允许创建初始管理员
- 有效凭据可在 `/login` 登录
- 无效或被禁用的账户无法登录
- 未认证访问受保护页面时重定向到登录页
- 管理员可在 `/settings/accounts` 管理账户
- 应用阻止移除或禁用最后一个启用的管理员

### 仪表板

- 显示活跃孩子及其当前余额
- 准确反映交易汇总数据
- 提供清晰的导航，方便进入交易录入、历史记录和报告流程

### 孩子管理

- 支持创建、编辑和禁用孩子
- 禁用的孩子不出现在新交易的默认选项中
- 禁用孩子的历史数据仍然可见

### 项目管理

- 支持创建、编辑和禁用奖励、惩罚和兑换项目
- 禁用的项目不出现在新交易的选择列表中
- 项目编辑或禁用后，历史交易渲染保持完整

### 交易录入

- 奖励条目增加余额
- 惩罚条目减少余额
- 仅在积分足够时兑换奖励才减少余额
- 交易存储孩子、项目、快照名称、积分、备注和发生时间

### 交易历史与编辑

- 交易可按孩子、类型和日期范围筛选
- 编辑交易后当前余额正确更新
- 每次编辑都创建一个 `TransactionRevision`
- 通过 `itemNameSnapshot` 保持历史项目名称可读

### 统计与导出

- 聚合统计数据与筛选后的交易总计一致
- CSV 导出与选定的交易集合匹配
- 导出的 CSV 文件中中文内容保持可读

## 部署预期

- 项目应保持可通过 Node.js 加 PostgreSQL 部署
- Docker 部署应通过附带的 `Dockerfile` 和 `compose.yaml` 保持支持
- 环境配置必须使用显式的 `DATABASE_URL`、`AUTH_SECRET` 及相关运行时设置
